## 🚀 เริ่มต้นอย่างรวดเร็ว

### ใช้งานออนไลน์ (แนะนำ)
1. เข้าไปที่ [https://twittervideodownloaderx.com/th](https://twittervideodownloaderx.com/th)
2. คอปปี้ลิงก์โพสต์เป้าหมาย (ตัวอย่าง: `https://x.com/user/status/123456789`)
3. วางลิงก์ในช่องป้อนข้อมูล → คลิกปุ่ม 「วิเคราะห์」
4. เลือกรูปแบบที่ต้องการ → บันทึกไฟล์ตามคำแนะนำของเบราว์เซอร์

### การปรับใช้สำหรับผู้พัฒนาระบบ (ติดตั้งในเครื่อง)
```bash
# โคลนรีพอซิทอรี
git clone https://github.com/your-repo/x-video-parser.git

# ติดตั้งแพ็กเกจที่จำเป็น
cd x-video-parser && npm install

# ตั้งค่าตัวแปรสภาพแวดล้อม (ไม่บังคับ)
cp .env.example .env

# เริ่มเซิร์ฟเวอร์สำหรับพัฒนา
npm run dev
```

> 💡 หมายเหตุ: โครงการนี้ใช้สถาปัตยกรรม Node.js + Express กรุณาอ่านคู่มือการปรับใช้โดยละเอียดได้ที่ `/docs/DEPLOY.md`

## 📚 คู่มือการใช้งาน

### ขั้นตอนการใช้งานพื้นฐาน
```
1. รับลิงก์โพสต์
   └─ เปิดโพสต์เป้าหมายบน X.com → คอปปี้ URL จากแถบที่อยู่ของเบราว์เซอร์

2. ส่งคำขอวิเคราะห์
   └─ วางลิงก์ในช่องป้อนข้อมูลของเครื่องมือ → คลิก 「ดาวน์โหลดวิดีโอ」

3. เลือกรูปแบบผลลัพธ์
   ├─ 🎬 ดาวน์โหลดเป็น MP4: บันทึกด้วยคุณภาพต้นฉบับ
   ├─ 🎵 แปลงเป็น MP3: สกัดแทร็กเสียง
   └─ 🎞 แปลงเป็น GIF: สร้างภาพเคลื่อนไหวสั้นๆ (แนะนำ: ไม่เกิน 15 วินาที)

4. บันทึกไฟล์
   └─ ทรัพยากรจะเปิดในแท็บใหม่ → คลิกขวา/เมนู → 「บันทึกเป็น」
```

### เคล็ดลับการใช้งานบนสมาร์ทโฟน
- iOS Safari: ปุ่มแชร์ → 「บันทึกในไฟล์」
- Android Chrome: กดวิดีโอค้างไว้ → 「ดาวน์โหลดวิดีโอ」
- กรณีวิดีโอเล่นอัตโนมัติ: คลิก `⋮` มุมขวาบนของผู้เล่น → เลือก 「ดาวน์โหลด」

## ❓ คำถามที่พบบ่อย

**ถาม: ไฟล์ที่ดาวน์โหลดจะบันทึกไว้ที่ไหน?**  
ตอบ: จะบันทึกไว้ในโฟลเดอร์ดาวน์โหลดที่กำหนดไว้ในการตั้งค่าเบราว์เซอร์ ท่านสามารถตรวจสอบหรือเปลี่ยนเส้นทางบันทึกได้ในหน้าตั้งค่าของเบราว์เซอร์

**ถาม: สามารถวิเคราะห์บัญชีส่วนตัวหรือเนื้อหาที่ถูกจำกัดได้หรือไม่?**  
ตอบ: ไม่ได้ เครื่องมือนี้รองรับเฉพาะโพสต์ที่ตั้งค่าเป็นสาธารณะเท่านั้น และเคารพการตั้งค่าสิทธิ์การเข้าถึงของเนื้อหาต้นฉบับ

**ถาม: คุณภาพภาพ/เสียงหลังการแปลงจะลดลงหรือไม่?**  
ตอบ: รูปแบบ MP4 จะรักษาคุณภาพต้นฉบับไว้ รูปแบบ MP3 ใช้การเข้ารหัสมาตรฐาน 128kbps ส่วนรูปแบบ GIF จะปรับอัตราเฟรมตามระยะเวลาการเล่น เพื่อสร้างสมดุลระหว่างขนาดไฟล์และความลื่นไหล

**ถาม: ประวัติการดาวน์โหลดหรือแคชจะถูกบันทึกไว้หรือไม่?**  
ตอบ: ไม่ถูกบันทึก ทรัพยากรทั้งหมดจะถูกส่งตรงไปยังอุปกรณ์ของผู้ใช้ผ่านพร็อกซีชั่วคราว เซิร์ฟเวอร์จะไม่เก็บข้อมูลคำขอหรือไฟล์มีเดียใดๆ ทั้งสิ้น

**ถาม: หากวิเคราะห์ล้มเหลวควรทำอย่างไร?**  
ตอบ: กรุณาตรวจสอบ ① ลิงก์โพสต์สาธารณะที่ใช้งานได้อยู่ ② ความเสถียรของเครือข่าย ③ ลองเปลี่ยนเบราว์เซอร์แล้วทดสอบใหม่ หากปัญหายังคงอยู่ กรุณารายงานผ่าน Issue ได้ตลอดเวลา
