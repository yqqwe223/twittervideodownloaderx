## 📋 परियोजना अवलोकन

यह परियोजना एक वेब-आधारित वीडियो विश्लेषण उपकरण है, जो X.com (पूर्व में Twitter) पर सार्वजनिक पोस्ट से वीडियो संसाधनों को सुरक्षित रूप से निकालने और विभिन्न प्रारूपों में परिवर्तन व डाउनलोड करने का विकल्प प्रदान करने के लिए डिज़ाइन किया गया है। किसी क्लाइंट सॉफ़्टवेयर इंस्टॉलेशन या खाता पंजीकरण की आवश्यकता नहीं: अपने ब्राउज़र के माध्यम से सीधे उपयोग करें।

> ⚠️ **महत्वपूर्ण सूचना**: यह उपकरण विशेष रूप से व्यक्तिगत शिक्षण, तकनीकी शोध और उचित सीमाओं के भीतर उपयोग के लिए डिज़ाइन किया गया है। कृपया [X प्लेटफ़ॉर्म डेवलपर समझौते](https://developer.twitter.com/en/developer-terms/agreement) और अपने देश में लागू कॉपीराइट कानूनों का पालन करें। तीसरे पक्ष के अधिकारों का उल्लंघन करने वाले उद्देश्यों के लिए इसके उपयोग पर सख्त प्रतिबंध है।

## ✨ मुख्य विशेषताएं

- 🔗 **लिंक विश्लेषण**: मानक X.com पोस्ट URL का समर्थन; वीडियो और एनिमेटेड GIF का स्वचालित पहचान
- 📥 **बहु-प्रारूप निर्यात**:
  - मूल गुणवत्ता में MP4 वीडियो
  - ऑडियो निष्कर्षण → MP3 प्रारूप
  - वीडियो क्लिप → एनिमेटेड GIF में परिवर्तन
- 🌍 **बहुभाषी इंटरफ़ेस**: हिंदी, अंग्रेजी, चीनी, जापानी, कोरियाई सहित कुल 16 भाषाओं का समर्थन
- 📱 **क्रॉस-प्लेटफ़ॉर्म संगतता**: Chrome / Firefox / Safari / Edge पर निर्बाध कार्य; मोबाइल उपकरणों और टैबलेट के लिए अनुकूलित अनुभव
- 🔒 **गोपनीयता सर्वोपरि**: कोई खाता पंजीकरण आवश्यक नहीं, कोई व्यक्तिगत डेटा संग्रह नहीं; विश्लेषण प्रक्रिया पूरी तरह से अनाम
- ⚡ **तेज़ प्रसंस्करण**: विश्लेषण औसतन 5 सेकंड से कम समय में पूरा; समवर्ती अनुरोधों का समर्थन

## 🚀 त्वरित आरंभ

### ऑनलाइन उपयोग (अनुशंसित)
1. [https://twittervideodownloaderx.com/hi](https://twittervideodownloaderx.com/hi) पर जाएं
2. लक्ष्य पोस्ट का लिंक कॉपी करें (उदाहरण: `https://x.com/user/status/123456789`)
3. लिंक को इनपुट फ़ील्ड में पेस्ट करें → 「विश्लेषण करें」बटन पर क्लिक करें
4. वांछित प्रारूप चुनें → ब्राउज़र के निर्देशों का पालन करते हुए फ़ाइल सहेजें

### स्थानीय तैनाती (डेवलपर्स के लिए)
```bash
# रिपॉजिटरी क्लोन करें
git clone https://github.com/your-repo/x-video-parser.git

# निर्भरताएं इंस्टॉल करें
cd x-video-parser && npm install

# पर्यावरण चर कॉन्फ़िगर करें (वैकल्पिक)
cp .env.example .env

# विकास सर्वर शुरू करें
npm run dev
```

> 💡 नोट: यह परियोजना Node.js + Express आधारित आर्किटेक्चर का उपयोग करती है। विस्तृत तैनाती दस्तावेज़ीकरण के लिए `/docs/DEPLOY.md` देखें

## 📚 उपयोग गाइड

### मूल कार्य प्रवाह
```
1. पोस्ट लिंक प्राप्त करें
   └─ X.com पर लक्ष्य पोस्ट खोलें → ब्राउज़र के एड्रेस बार से URL कॉपी करें

2. विश्लेषण अनुरोध भेजें
   └─ लिंक को उपकरण के इनपुट फ़ील्ड में पेस्ट करें → 「वीडियो डाउनलोड करें」पर क्लिक करें

3. आउटपुट प्रारूप चुनें
   ├─ 🎬 MP4 के रूप में डाउनलोड करें: मूल गुणवत्ता के साथ सहेजें
   ├─ 🎵 MP3 में परिवर्तित करें: ऑडियो ट्रैक निकालें
   └─ 🎞 GIF में परिवर्तित करें: छोटी एनिमेशन बनाएं (अनुशंसित: ≤15 सेकंड)

4. फ़ाइल सहेजें
   └─ संसाधन नए टैब में खुलेगा → राइट-क्लिक/मेनू → 「इस रूप में सहेजें」
```

### मोबाइल उपकरणों पर उपयोग के लिए टिप्स
- iOS Safari: साझा करें बटन → 「फ़ाइलों में सहेजें」
- Android Chrome: वीडियो पर लंबे समय तक दबाएं → 「वीडियो डाउनलोड करें」
- यदि वीडियो स्वचालित रूप से चल रहा है: प्लेयर के ऊपरी दाएं कोने में `⋮` पर क्लिक करें → 「डाउनलोड」चुनें

## ❓ अक्सर पूछे जाने वाले प्रश्न

**प्र: डाउनलोड की गई फ़ाइलें कहाँ सहेजी जाती हैं?**  
उ: फ़ाइलें आपके ब्राउज़र में कॉन्फ़िगर किए गए डाउनलोड फ़ोल्डर में सहेजी जाती हैं। आप ब्राउज़र सेटिंग्स में इस पथ की जांच कर सकते हैं या बदल सकते हैं।

**प्र: क्या मैं निजी खातों या प्रतिबंधित सामग्री का विश्लेषण कर सकता हूँ?**  
उ: नहीं। यह उपकरण केवल सार्वजनिक रूप से सेट किए गए पोस्ट के साथ काम करता है और मूल सामग्री की पहुंच सेटिंग्स का सम्मान करता है।

**प्र: क्या परिवर्तन के बाद छवि/ऑडियो गुणवत्ता कम हो जाती है?**  
उ: MP3 प्रारूप 128 kbps मानक एन्कोडिंग का उपयोग करता है। GIF प्रारूप फ़ाइल आकार और सुचारू प्रदर्शन के बीच संतुलन बनाने के लिए अवधि के अनुसार फ़्रेम दर को अनुकूलित करता है।

**प्र: क्या डाउनलोड इतिहास या कैश सहेजा जाता है?**  
उ: नहीं। सभी संसाधन अस्थायी प्रॉक्सी के माध्यम से सीधे उपयोगकर्ता के डिवाइस पर प्रसारित किए जाते हैं; सर्वर किसी भी अनुरोध या मीडिया फ़ाइल को संग्रहीत नहीं करता है।

**प्र: यदि विश्लेषण विफल हो जाए तो क्या करें?**  
उ: कृपया जांचें: ① क्या लिंक एक वैध सार्वजनिक पोस्ट से संबंधित है ② क्या आपका इंटरनेट कनेक्शन स्थिर है ③ किसी अन्य ब्राउज़र का प्रयास करें। यदि समस्या बनी रहती है, तो Issue के माध्यम से रिपोर्ट करने में संकोच न करें।
