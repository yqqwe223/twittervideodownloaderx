# 🐦 X.com 동영상 분석 도구

> 가볍고 빠른, 다기능을 갖춘 X (구 Twitter) 동영상 콘텐츠 추출 도구

[🌐 온라인 데모](https://twittervideodownloaderx.com/ko) • [📝 사용 가이드](#-사용-가이드) • [❓ 자주 묻는 질문](#-자주-묻는-질문)

## 📋 프로젝트 개요

본 프로젝트는 X.com (구 Twitter) 의 공개 게시물에서 동영상 리소스를 안전하게 추출하고, 다양한 포맷으로 변환·저장할 수 있도록 지원하는 웹 기반 분석 도구입니다. 별도 클라이언트 설치나 계정 등록 없이, 브라우저에서 즉시 이용하실 수 있습니다.

> ⚠️ **중요 안내**: 본 도구는 개인 학습, 기술 연구, 합리적인 이용 범위 내에서만 사용하시기 바랍니다. [X 플랫폼 개발자 이용약관](https://developer.twitter.com/en/developer-terms/agreement) 및 각국 저작권법을 준수하여, 타인의 권리를 침해하는 목적으로의 사용은 엄격히 금지됩니다.

---

## ✨ 주요 기능

- 🔗 **링크 분석**: 표준 X.com 게시물 URL 지원, 동영상/애니메이션 GIF 자동 식별
- 📥 **다양한 내보내기 형식**:
  - 원본 화질 MP4 동영상
  - 오디오 추출 → MP3 형식
  - 동영상 클립 → GIF 애니메이션 변환
- 🌍 **다국어 인터페이스**: 한국어, 영어, 중국어, 일본어 등 16 개 언어 지원
- 📱 **크로스 플랫폼 호환**: Chrome / Firefox / Safari / Edge 대응, 스마트폰·태블릿에서도 쾌적한 사용 경험
- 🔒 **프라이버시 우선**: 계정 등록 불필요, 사용자 정보 수집 없음, 분석 프로세스 완전 익명화
- ⚡ **고속 처리**: 평균 5 초 이내 분석 완료, 병렬 요청 지원

## 🚀 빠른 시작

### 온라인 이용 (권장)
1. [https://twittervideodownloaderx.com/ko](https://twittervideodownloaderx.com/ko) 에 접속
2. 대상 게시물 링크 복사 (예: `https://x.com/user/status/123456789`)
3. 입력창에 붙여넣기 → 「분석」버튼 클릭
4. 원하는 형식 선택 → 브라우저 안내에 따라 파일 저장

### 로컬 개발자용 배포
```bash
# 저장소 클론
git clone https://github.com/your-repo/x-video-parser.git

# 종속성 설치
cd x-video-parser && npm install

# 환경 변수 설정 (선택사항)
cp .env.example .env

# 개발 서버 실행
npm run dev
```

> 💡 참고: 본 프로젝트는 Node.js + Express 아키텍처를 기반으로 합니다. 상세 배포 절차는 `/docs/DEPLOY.md` 를 참조해 주세요.

## 📚 사용 가이드

### 기본 조작 흐름
```
1. 게시물 링크 취득
   └─ X.com 에서 대상 게시물 열기 → 브라우저 주소창에서 URL 복사

2. 분석 요청 전송
   └─ 도구 입력창에 링크 붙여넣기 → 「동영상 다운로드」클릭

3. 출력 형식 선택
   ├─ 🎬 MP4 로 다운로드: 원본 화질로 저장
   ├─ 🎵 MP3 로 변환: 오디오 트랙 추출
   └─ 🎞 GIF 로 변환: 짧은 애니메이션 생성 (권장: 15 초 이내)

4. 파일 저장
   └─ 새 탭에서 리소스 열림 → 우클릭/메뉴 → 「다른 이름으로 저장」
```

### 스마트폰 이용 팁
- iOS Safari: 공유 버튼 → 「파일에 저장」
- Android Chrome: 동영상 길게 누르기 → 「동영상 다운로드」
- 자동 재생 시: 플레이어 우측 상단 `⋮` → 「다운로드」선택

## ❓ 자주 묻는 질문

**Q: 다운로드한 파일은 어디에 저장되나요?**  
A: 브라우저 설정에 지정된 다운로드 폴더에 저장됩니다. 브라우저 설정 화면에서 저장 경로를 확인하거나 변경할 수 있습니다.

**Q: 비공개 계정이나 제한된 콘텐츠도 분석할 수 있나요?**  
A: 아니요. 본 도구는 공개 설정된 게시물만 지원하며, 원본 콘텐츠의 접근 권한 설정을 존중합니다.

**Q: 변환 후 화질/음질이 저하되나요?**  
A: MP4 형식은 원본 화질을 유지합니다. MP3 는 128kbps 표준 인코딩을, GIF 는 재생 시간에 따라 프레임 레이트를 최적화하여 파일 크기와 부드러움의 균형을 맞춥니다.

**Q: 다운로드 이력이나 캐시가 저장되나요?**  
A: 저장되지 않습니다. 모든 리소스는 임시 프록시를 거쳐 직접 사용자 기기로 전송되며, 서버 측에서는 요청 내용이나 미디어 파일을 일절 보관하지 않습니다.

**Q: 분석에 실패한 경우 어떻게 해야 하나요?**  
A: ① 유효한 공개 게시물 링크인지 확인 ② 네트워크 환경 안정성 확인 ③ 브라우저를 변경하여 재시도 해 주세요. 문제가 지속될 경우, Issue 를 통해 자유롭게 보고해 주시기 바랍니다.
