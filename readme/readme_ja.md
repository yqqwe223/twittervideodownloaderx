# 🐦 X.com 動画解析ツール

> 軽量・高速・多機能な X (Twitter) 動画コンテンツ抽出ツール

[🌐 オンラインデモ](https://twittervideodownloaderx.com/ja) • [📝 使い方ガイド](#-使い方ガイド) • [❓ よくある質問](#-よくある質問)

## 📋 プロジェクト概要

本プロジェクトは、X.com（旧 Twitter）の公開投稿から動画リソースを安全に抽出し、複数形式への変換・保存をサポートする Web ベースの解析ツールです。クライアント不要・登録不要で、ブラウザからすぐに利用できます。

> ⚠️ **重要なお知らせ**：本ツールは個人学習・技術研究・合理的な利用範囲でのみご使用ください。[X プラットフォーム開発者規約](https://developer.twitter.com/en/developer-terms/agreement) 及び各国の著作権法を遵守し、他者の権利を侵害する目的での利用は固くお断りいたします。

---

## ✨ 主な機能

- 🔗 **リンク解析**：標準的な X.com 投稿 URL に対応、動画/アニメーション GIF を自動識別
- 📥 **多形式エクスポート**：
  - オリジナル画質の MP4 動画
  - 音声抽出 → MP3 形式
  - 動画クリップ → GIF アニメーション変換
- 🌍 **多言語インターフェース**：日本語・英語・中国語・韓国語など 16 言語をサポート
- 📱 **クロスプラットフォーム**：Chrome / Firefox / Safari / Edge 対応、スマートフォン・タブレットでも快適操作
- 🔒 **プライバシー重視**：アカウント登録不要、ユーザー情報収集なし、解析プロセスは完全匿名
- ⚡ **高速処理**：平均 5 秒以内で解析完了、並列リクエスト対応

## 🚀 クイックスタート

### オンライン利用（推奨）
1. [https://twittervideodownloaderx.com/ja](https://twittervideodownloaderx.com/ja) にアクセス
2. 対象の投稿リンクをコピー（例：`https://x.com/user/status/123456789`）
3. 入力欄に貼り付け →「解析」ボタンをクリック
4. 希望の形式を選択 → ブラウザの指示に従ってファイルを保存

### ローカル開発者向けデプロイ
```bash
# リポジトリのクローン
git clone https://github.com/your-repo/x-video-parser.git

# 依存パッケージのインストール
cd x-video-parser && npm install

# 環境変数の設定（任意）
cp .env.example .env

# 開発サーバーの起動
npm run dev
```

> 💡 補足：本プロジェクトは Node.js + Express 架构を採用。詳細なデプロイ手順は `/docs/DEPLOY.md` をご参照ください。

## 📚 使い方ガイド

### 基本操作フロー
```
1. 投稿リンクの取得
   └─ X.com 上で対象投稿を開く → ブラウザのアドレスバーから URL をコピー

2. 解析リクエストの送信
   └─ ツールの入力欄にリンクを貼り付け →「動画をダウンロード」をクリック

3. 出力形式の選択
   ├─ 🎬 MP4 でダウンロード：オリジナル画質で保存
   ├─ 🎵 MP3 に変換：音声トラックを抽出
   └─ 🎞 GIF に変換：短いアニメーションを作成（推奨：15 秒以内）

4. ファイルの保存
   └─ 新規タブでリソースが開く → 右クリック/メニュー →「名前を付けて保存」
```

### スマートフォン利用時のヒント
- iOS Safari：共有ボタン →「ファイルに保存」
- Android Chrome：動画を長押し →「動画をダウンロード」
- 自動再生される場合：プレーヤー右上の `⋮` →「ダウンロード」を選択

## ❓ よくある質問

**Q：ダウンロードしたファイルはどこに保存されますか？**  
A：ブラウザの設定で指定されたダウンロードフォルダに保存されます。ブラウザの設定画面から保存先を確認・変更可能です。

**Q：非公開アカウントや制限付きコンテンツは解析できますか？**  
A：いいえ。本ツールは公開設定の投稿のみ対応しており、元のコンテンツのアクセス権限設定を尊重します。

**Q：変換後の画質・音質は劣化しますか？**  
A：MP4 形式はオリジナル画質を維持します。MP3 は 128kbps の標準エンコード、GIF は再生時間に応じてフレームレートを最適化し、ファイルサイズと滑らかさのバランスを取ります。

**Q：ダウンロード履歴やキャッシュは保存されますか？**  
A：保存されません。すべてのリソースは一時プロキシ経由で直接ユーザー端末へ転送され、サーバー側ではリクエスト内容やメディアファイルは一切保持しません。

**Q：解析に失敗した場合はどうすればいいですか？**  
A：① 有効な公開投稿リンクか確認 ② ネットワーク環境の安定性を確認 ③ ブラウザを変更して再試行 してください。問題が解決しない場合は、Issue よりお気軽にご報告ください。
